Apostle - overview

16 May 2022

Apostle is a reliable, secure and scalable way to publish, consume and manage API’s.It ensures optimal performance of the API’s, tracks and enforces usage, authentication, and more. This topic provides an overview of Apostle, including register your API, create group and monitor usage. For more information on specific topic please see relative link on sidebar.

In this topic we will walk you through the process of registering your first API and monitoring its usage.

Let’s do it step by step.

Step One : Register your api

  1. Click on project on the left sidebar.

  2. If you don’t see you project listed, you need to register it on Project Management System, and if you don’t have access to that system, please contact administrator for registration.

  3. Click on Add API.

For more info on the fields click here

Step two : Create a group

  1. Click on add group on client page

  2. Click on select API from project list. You can select api from multiple projects.

  3. Select the APIs and click on next.

  4. Enter group name and user email for this group.

  5. Select package and add grace calls if it applies.

  6. Generate public key and save

For more info on fields click here

Step three : Create package [ only necessary if you want to use custom pacakge ]

  1. Click on packages on left sidebar.

  2. Click on add package.

  3. Fill out necessary details.

For more info on fields click here.
