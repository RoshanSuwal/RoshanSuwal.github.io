What is a package

16 May 2022

Packages in Apostle are subscription based. Each package has time interval based quota which defines how many API calls are allowed to pass to API server.Each group is used with a subscription package and every API inside the group can be configured to use different package which are called personal packages.So, if there are 3 APIs in a group then as per requirement each API can use different packages with different quota and expiration policy.

Creating a Package

Fields description

1. Name : Name of the group.

2. Total API calls : This value is the maximum allowed calls for any group that is using this package.
After this value is reached grace calls, if defined, will start and after grace calls are exhausted, public key is declared expired.

3. Duration : Currently supported Durations are :
. Minute
. Hour
. Day
. Month
. Year

We can define allowed calls for each duration and all extra calls will be discarded by the Validator, thus protecting API server from request overloading.
